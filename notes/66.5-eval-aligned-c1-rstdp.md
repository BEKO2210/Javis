# Iter 66.5 — Eval-aligned R-STDP for C1 (minimal Path-1 fix)

**Status: ENTRY (pre-registration). No measurements, no
implementation yet. The hypothesis, locked seed set, locked
acceptance gate, locked metric, and locked CLI surface are
committed in this file *before* any code is written. They may
not be relaxed post-hoc.**

> **Core sentence.** Iter-66 step-7.5 falsified iter-66's
> training assumption: R-STDP under double-clamp (R2-target +
> C1-target + M_target) trains the (canonical R2 target SDR)
> → (canonical C1 target SDR) pathway, but the canonical R2
> target SDR does not appear during cue-only eval (iter-65 R2
> readout = chance). Iter-66.5 implements the deep-research
> Path-1 fix: drop the R2 target clamp during the teacher
> window; let R2 fire its actual cue-driven response; have
> R-STDP align (eval-time R2 cue pattern) → (canonical C1
> target). No new architecture, no new metric, no new
> plasticity rules.

## Why iter-66.5 (and not directly iter-67)

iter-66's step 7.5 verdict (notes/66 §"Step 7.5") locked the
diagnosis as a **pattern mismatch**, not a silence/wiring
failure: the R-STDP path is alive, weights are moving every
epoch, the M_target gate fires on every trial, the C1 clamp
hits 100% of the canonical target cells. What fails is the
*alignment*: training-time pre-synaptic substrate is the
clamped canonical R2 target SDR, eval-time pre-synaptic
substrate is R2's natural cue-driven response, and the two
patterns have ≈ chance overlap because R2 itself has not
learned to bind cue → canonical target SDR (iter-65 falsified
that branch at 8 seeds).

iter-66.5 is the smallest possible scope correction to that
single mismatch. It does not modify the C1 layer wiring, the
metric, the seed set, the acceptance matrix structure, or any
plasticity rule. It only changes *what R2 fires during the
teacher window when C1 supervision is active*.

## Pre-registered hypothesis

> On the iter-63-baseline brain configuration *with* the
> iter-66 C1 layer (1000 LIF neurons, k=20 sparsity,
> R2-E → C1 plastic projection, fan-out per the iter-66 ENTRY)
> AND the iter-66.5 modified teacher schedule (R2 target
> clamp DROPPED during Phase 4 when c1.enabled, C1 target
> clamp + M_target = +1 retained), 32 epochs of training on
> seed 42 produce a *measurable, non-trivial, monotonically
> rising* `c1_target_top3_overlap` curve over the per-epoch
> readouts.

This hypothesis is *strictly weaker* than iter-66's. iter-66
required `Δ̄ > 0.05 AND n_pos ≥ 7/8` for a (B) Robust verdict.
iter-66.5 requires only "single-seed rising curve > 0 at 32
epochs" for the gate that re-opens discussion of a full run.
The strict 8-seed acceptance matrix from notes/66 §"Locked
acceptance matrix" carries over verbatim *for the eventual
8-seed verdict run*, which iter-66.5 does NOT execute on its
own — that decision waits for the rising-curve gate.


The semantic of the change: when C1 supervision is active, R2
is NOT supervised by the canonical-target clamp. R-STDP on
R2-E → C1 then learns "what R2 actually does for this cue" →
"the canonical C1 target pattern". The canonical R2 target
SDR ceases to be a training target on the R2 side; it lives
only as the C1-side teacher signal.

## Locked methodological commitments

1. **No new plasticity rule.** The change is purely scheduling
   (which cells get the clamp current during Phase 4). All
   existing rules (STDP, iSTDP, R-STDP, homeostasis, intrinsic,
   reward) operate exactly as they did in iter-66.
2. **No new architecture.** C1 layer wiring, R2-E → C1
   projection, M_target modulator semantics, all unchanged.
3. **No new metric.** `c1_target_top3_overlap` (and the
   companion R2 `target_top3_overlap`) are the only metrics.
4. **No iter-66 acceptance-matrix relaxation.** The (A) / (B)
   / (C) / (D) classification carries over verbatim *for the
   eventual main run*. iter-66.5 itself does not do an 8-seed
   verdict — it does a single-seed gate first.
5. **No 8-seed run before the rising-curve gate is met.**
6. **iter-65/iter-66 numerics preserved when c1.enabled=false.**
   The ten existing reward_bench snapshot tests must continue
   to pass bit-identically. iter-66 already established this
   contract; iter-66.5 maintains it.

## BTSP (Mechanism M5) — iter-67 fallback, not iter-66.5

BTSP (Bittner 2017 / Magee & Grienberger 2020) replaces R-STDP
with a one-shot eligibility window. It is the recommended
fallback if Path 1 + Path 2 do NOT produce a rising curve at
32 ep on seed 42. BTSP requires a new plasticity rule in
snn-core, which is the largest scope of the three follow-up
paths. Pre-registration for BTSP would be a separate iter-67
ENTRY *only after* iter-66.5 / iter-66.6 falsify the simpler
fixes.

## Locked primary metric

`c1_target_top3_overlap` — unchanged from iter-66 ENTRY. The
per-epoch readout is computed exactly as in iter-66:

1. Build a per-epoch decoder dictionary from C1 fingerprints
   of every vocab word (run each word's cue, capture C1
   activity, kWTA top-K=20, store as that word's fingerprint).
2. For every real (cue, target) pair, present the cue alone,
   capture C1's decoded top-3, score 1 if the canonical target
   word is in the top-3 else 0.
3. `c1_target_top3_overlap_epoch = top3_hits / n_real_pairs`.

`c1_target_top3_overlap` = mean over all 32 epochs (same
estimator iter-65 / iter-66 used).

The companion R2 `target_top3_overlap` is reported alongside
for cross-iter comparability — iter-65's R2 number at the
same config was 0.0396 ± 0.025 across 8 seeds; the iter-66
step-7.5 smoke showed 0.0273 (1 seed × 8 ep) and 0.0312
(1 seed × 4 ep) at the locked config. iter-66.5 *may* shift
R2's reading because the R2 target clamp is dropped on
c1-active trials — the diagnostic must report this drift
without re-classifying it as a verdict result (iter-65 already
locked the R2 reading at chance).

The iter-66 step-7.5 diagnostic instrumentation
(`--c1-diagnostic`) carries over unchanged; iter-66.5 smoke
*requires* it on so the per-epoch curve is visible in the log.

## Locked acceptance gate (single-seed rising curve)

The seed-42 single-seed run at 32 epochs yields a vector of
per-epoch readings `c1_top3[0..32]`. Bekos's locked gate:

| Outcome | Per-epoch pattern | Action |
| --- | --- | --- |
| **(P1.A) Rising — clear** | mean over last 8 epochs `≥ 0.05` AND mean over first 8 epochs `< 0.02` AND no contiguous-3-epoch dip below first-8-mean | iter-66.5 cleared. 8-seed × 32-ep main run becomes legitimate to *propose* (not auto-launched; Bekos must explicitly approve) under the iter-66 acceptance matrix. |
| **(P1.B) Rising — weak** | mean over last 8 epochs `> 0.0` AND `> mean over first 8 epochs` AND not (P1.A) | iter-66.5 partially passes. iter-66.6 = Path 1 + Path 2 (homeostasis-off) single-seed smoke at the same config. |
| **(P1.C) Flat at zero** | last-8-epoch mean `= 0` (or `≤ 0.001`) | iter-66.5 falsifies Path 1 alone. iter-66.6 = Path 1 + Path 2 single-seed smoke; if that also fails, iter-67 = BTSP pre-registration. |
| **(P1.D) Worse than iter-66** | last-8-epoch mean `< 0` impossible at this metric (it's `≥ 0` by construction) — but: c1_active_frac during teacher drops below 0.95, OR clamp_eff drops below 0.95, OR R2 readout drops below 0.01 (i.e. dropping the R2 clamp also breaks R2 itself) | iter-66.5 has a regression. Roll back; investigate; do NOT proceed to Path 2 or BTSP until the regression is understood. |

The diagnostic instrumentation must report enough to
discriminate (P1.A/B/C/D) on a single seed-42 run. The
existing `--c1-diagnostic` per-epoch line covers all four
cases (top3_c1, c1_active_frac, clamp_eff, top3_r2 are all
already logged).

## Locked CLI surface

```sh
# Smoke — single-seed rising-curve gate (the only run iter-66.5
# pre-authorises). 32 epochs × seed 42 with the schedule fix on.
cargo run --release -p eval --example reward_benchmark -- \
  --c1-readout --c1-diagnostic \
  --c1-teacher-strength 1.0 \
  --c1-eval-aligned-rstdp \
  --seeds 42 --epochs 32 \
  --teacher-forcing --target-clamp-strength 500 --teacher-ms 40 \
  --corpus-vocab 64 --dg-bridge --plasticity-off-during-eval \
  --decorrelated-init
```

`--c1-eval-aligned-rstdp` is the new boolean flag that gates
the Path-1 schedule change. Default off ⇒ iter-66 behaviour
verbatim (every existing snapshot test still passes
bit-identically). When set together with `--c1-readout`, the
teacher Phase 4 omits the R2 target SDR from the clamp
indices for that brain.

8-seed verdict CLI is *not* listed here. It will be appended
to this file only after the seed-42 gate clears and Bekos
approves a follow-up.

## Implementation sequence (locked)

1. **Step 1 — `eval_aligned_rstdp: bool` field in `C1Config`**
   (default false). Plumb through `TeacherForcingConfig` Default
   + ::off() initialisers.
2. **Step 2 — Conditional clamp construction in
   `run_teacher_trial`.** When `cfg.c1.enabled &&
   cfg.c1.eval_aligned_rstdp`, build `combined_clamp` from
   `c1_target_sdr` only (omit `target_r2_sdr`). When false,
   bit-identical to iter-66.
3. **Step 3 — CLI flag** `--c1-eval-aligned-rstdp` +
   `C1Config` initialiser plumbing.
4. **Step 4 — Smoke** (1 seed × 32 ep with `--c1-diagnostic
   --c1-eval-aligned-rstdp`). Apply the (P1.A/B/C/D) gate.
5. **Step 5 — Verdict commit** (this file appended with the
   per-epoch curve, the gate classification, and the
   conditional follow-up plan).

Steps 1–3 are committed together as the implementation;
step 4 is a smoke commit; step 5 is the verdict commit.
